Arquitetura do Sistema

O sistema implementa o conceito de Gêmeo Digital (Digital Twin), onde o estado de um dispositivo físico é espelhado em tempo real em uma interface virtual e vice-versa. Além disso, atua como gateway M2M (Machine-to-Machine) para controle de um nó remoto.

  • Nó Controlador: Hospeda servidor Web e WebSocket, gerencia I/O físico (botões/display) e publica comandos MQTT.
  • Nó Receptor: Dispositivo passivo que assina tópicos MQTT e atualiza seu hardware (LED RGB).
  • Broker MQTT: Intermediário TCP/IP para desacoplamento dos nós.
  • Cliente Web: Interface HTML5/JS embarcada na memória do Controlador.

Protocolo de Dados (JSON)

O mesmo payload JSON é utilizado tanto na comunicação via WebSocket quanto via MQTT para garantir consistência de dados.

{
  "r": 255,    // Canal Vermelho (0-255)
  "g": 128,    // Canal Verde (0-255)
  "b": 0,      // Canal Azul (0-255)
  "bri": 100,  // Brilho Global (0-255)
  "pwr": 1     // Estado de Energia: 1 (Ligado), 0 (Desligado)
}

Firmware Controlador: Gestão de Estado

O estado do sistema é centralizado em uma estrutura global, garantindo que qualquer alteração (seja via botão físico ou via web) reflita na mesma fonte de verdade.

struct Estado {
  int r = 50;
  int g = 50;
  int b = 50;
  int brilho = 15;
  bool ligado = true;
  int menuIndex = 0; // 0:R, 1:G, 2:B, 3:Brilho
};

Firmware Controlador: Rede e Sincronia

void atualizarRede()

Esta função é responsável por propagar o estado atual para todos os clientes conectados.

  • WebSocket Broadcast: Envia o JSON para todos os navegadores conectados via webSocket.broadcastTXT(json).
  • MQTT Publish: Publica o JSON no tópico laboratorio/led. Caso o sistema esteja "desligado" (pwr=0), publica um JSON com cores zeradas para apagar o LED remoto, mas mantém o estado interno preservado.
  • Rate Limiting: A função é controlada por um temporizador (INTERVALO_MQTT = 100ms) no loop principal para evitar congestionamento da rede durante ajustes rápidos nos sliders.

void webSocketEvent(...)

Callback acionado ao receber dados do navegador.

  • Deserializa o JSON recebido.
  • Atualiza a struct sistema.
  • Seta flags precisaAtualizarVisual e precisaAtualizarRede para forçar atualização do OLED e reenvio MQTT.
  • Reseta lastMqttSend para garantir resposta imediata.

Firmware Controlador: Entradas Físicas

void lerBotoesAjuste()

Implementa lógica de incremento/decremento com aceleração.

  • Clique Simples: Altera o valor em 1 unidade.
  • Hold (Segurar): Se o botão for mantido pressionado por mais de 400ms, entra em modo de repetição rápida, alterando o valor em passos de 5 unidades a cada 50ms.
  • Usa a variável menuIndex para determinar qual campo da struct (r, g, b ou brilho) será afetado via ponteiros.

void lerBotaoPower()

Alterna o booleano sistema.ligado. Quando desligado, o OLED exibe "OFF" e o controlador envia comando de desligamento via MQTT, mas retém as configurações de cor na memória para quando for religado.

Firmware Controlador: Interface OLED

void desenharTela()

Renderiza a interface gráfica baseada no estado atual.

  • Cabeçalho: Exibe título e indicador de conexão MQTT (círculo cheio = conectado).
  • Lista de Parâmetros: Itera sobre R, G, B e BRI.
  • Cursor: Desenha o símbolo ">" ao lado do item selecionado por menuIndex.
  • Barras de Progresso: Desenha retângulos preenchidos proporcionalmente ao valor (0-255 mapeado para pixels) para feedback visual imediato.

Firmware Receptor: Lógica de Atuação

O firmware do receptor é minimalista, atuando como um escravo MQTT.

void callback(...)

Função executada a cada mensagem recebida no tópico laboratorio/led.

  1. Recebe o payload byte*.
  2. Converte para StaticJsonDocument.
  3. Extrai R, G, B e Bri.
  4. Aplica ao hardware via strip.setBrightness(bri) e strip.setLedColorData(...).
  5. Chama strip.show() para atualizar o LED WS2812.

Interface Web (Client-Side)

A página HTML é armazenada na memória flash do ESP32 (variável html_page) e servida via rota raiz /.

JavaScript Embarcado

  • WebSocket Init: Conecta em ws://IP_ESP32:81/. Gerencia reconexão automática em caso de queda.
  • Sincronia (Inbound): Ao receber mensagem do servidor (websocket.onmessage), atualiza os sliders HTML e a cor da div de preview, garantindo que mudanças físicas movam os sliders virtuais.
  • Controle (Outbound): A função uiChange() é chamada no evento oninput dos sliders. Ela monta o JSON e envia via WebSocket. Possui um limitador de taxa (throttle) de 50ms para não saturar o ESP32.
  • Preview de Cor: Converte os valores RGB para Hexadecimal e atualiza o background da interface dinamicamente.
