Arquitetura do Sistema

O sistema consiste em um firmware embarcado no microcontrolador ESP32-S3 que gerencia um LED RGB endereçável (WS2812) integrado à placa. A interação com o usuário é realizada através de uma interface de linha de comando (CLI) via porta Serial (UART).

  • Entrada: Comandos ASCII enviados via Monitor Serial.
  • Processamento: Interpretação de strings e máquina de estados simples.
  • Saída: Sinal digital (RMT) para o LED WS2812 e feedback textual na Serial.

Firmware: Variáveis e Estruturas

Matriz de Cores

As cores disponíveis são armazenadas em uma matriz bidimensional constante, facilitando a seleção por índice.

uint8_t colors[7][3] = {
  {255, 0, 0},    // 0: Vermelho
  {0, 255, 0},    // 1: Verde
  {0, 0, 255},    // 2: Azul
  {255, 255, 0},  // 3: Amarelo
  {255, 0, 255},  // 4: Magenta
  {0, 255, 255},  // 5: Ciano
  {255, 255, 255} // 6: Branco
};

Firmware: Funções Principais

void setup()

Inicializa os periféricos e define o estado inicial.

  • Inicializa a Serial a 115200 baud.
  • Inicializa o objeto strip.
  • Define o brilho inicial.
  • Chama atualizarLED() e exibirMenu() pela primeira vez.

void loop()

Executa o ciclo de leitura e interpretação de comandos.

  1. Verifica se há dados disponíveis no buffer Serial (Serial.available()).
  2. Lê a string recebida até encontrar uma quebra de linha (\n).
  3. Remove espaços em branco com trim().
  4. Compara a string recebida com as opções válidas ("1" a "9").
  5. Executa a ação correspondente e atualiza o menu.

Firmware: Funções Auxiliares

void exibirMenu()

Envia via Serial o texto formatado do menu de opções, incluindo o status atual (cor selecionada e nível de brilho).

void atualizarLED()

Aplica as configurações ao hardware.

  • Chama strip.setLedColorData() passando o índice do LED (0) e os valores R, G, B da matriz colors baseada em selectedColor.
  • Chama strip.show() para enviar o sinal RMT.

void apagarLED()

Envia a cor preta (0, 0, 0) para o LED, efetivamente desligando-o, mas mantendo a variável de estado selectedColor inalterada para futuro acionamento.

Protocolo de Interface Serial

A comunicação ocorre em modo texto ASCII.

Configuração do Terminal

  • Baud Rate: 115200
  • Line Ending: Newline (NL) ou Both (NL & CR). O código espera o caractere \n para processar o comando.
