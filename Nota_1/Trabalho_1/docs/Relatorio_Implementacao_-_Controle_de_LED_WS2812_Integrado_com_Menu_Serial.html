Objetivo e Escopo

O objetivo deste primeiro trabalho foi familiarizar-se com o ambiente de desenvolvimento para o ESP32-S3 e o controle de periféricos integrados. O foco específico foi o acionamento do LED RGB (WS2812) presente na placa de desenvolvimento, utilizando comunicação Serial para interação com o usuário.

O sistema deveria permitir a seleção de cores pré-definidas, o ajuste da intensidade luminosa (brilho) e o desligamento do LED, tudo através de comandos de texto.

Lógica de Controle

Para controlar o LED WS2812, que utiliza um protocolo de comunicação de um fio sensível ao tempo (timing-sensitive), utilizou-se a biblioteca ESP32_WS2812_Lib. Esta biblioteca abstrai a complexidade do uso do periférico RMT (Remote Control Transceiver) do ESP32.

Estrutura de Dados

Foi implementada uma matriz bidimensional uint8_t colors[7][3] contendo os valores RGB para as 7 cores principais (Vermelho, Verde, Azul, Amarelo, Magenta, Ciano, Branco). Isso permite que o código selecione a cor baseada apenas em um índice inteiro, simplificando a lógica do menu.

Manipulação Serial e Menu

A interação com o usuário foi construída baseada em um loop de verificação de buffer serial.

Parsing de Comandos

O comando Serial.readStringUntil('\n') foi utilizado para capturar a entrada do usuário. Isso exige que o Monitor Serial esteja configurado para enviar o caractere de nova linha (Newline) ao final do envio.

Fluxo de Bloqueio (Opção 8)

Uma particularidade da implementação é a opção de ajuste de brilho (Opção 8). Ao ser selecionada, o fluxo do programa entra em um loop while (Serial.available() == 0). Isso torna o sistema bloqueante temporariamente, aguardando obrigatoriamente que o usuário digite o valor do brilho (0-255) antes de retornar ao menu principal.

Testes Funcionais

O sistema foi testado enviando sequências de comandos via Monitor Serial do Arduino IDE.

  • Troca de Cores: A resposta do LED foi imediata ao envio dos números 1 a 7, com as cores correspondendo corretamente aos valores RGB definidos na matriz.
  • Ajuste de Brilho: O envio do comando 8 seguido de valores extremos (10 e 255) demonstrou a capacidade de dimerização do LED sem alteração da cor base (crominância).
  • Validação de Entrada: O uso de Serial.parseInt() na rotina de brilho filtrou corretamente caracteres não numéricos.

Conclusão

O trabalho cumpriu com êxito o objetivo de introdução à plataforma ESP32-S3. A utilização do LED integrado (GPIO 48) eliminou a necessidade de circuitos externos, permitindo foco na lógica de firmware e comunicação Serial. A implementação demonstrou como criar interfaces de usuário simples baseadas em texto para depuração e controle de dispositivos embarcados.
